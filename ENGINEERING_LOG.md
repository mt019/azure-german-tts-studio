# 工程日誌 (Engineering Log)

這個檔案用來紀錄本專案的功能調整、重構與實驗筆記，避免把 `README.md` 弄得太長、太複雜。  
`README.md` 只保留「怎麼用」與「環境設定」等必備資訊；任何細節優化都寫在這裡。

---

## 2025-12-02

- 新增：`ENGINEERING_LOG.md`，作為之後所有優化與調整的紀錄位置。
- 調整：`clean_markdown` / `split_sentences`
  - 清洗 Markdown 時不再把所有行接成一行，而是盡量保留原始換行。
  - 分句原本為「先依行拆，再在行內依標點切句」，確保原文本有換行的地方在結果中也一定會換行。
- 調整：YouTube 說明欄模板 `YOUTUBE_DESCRIPTION_TEMPLATES`
  - 確保所有模板都包含 Language Reactor 插件的學習小提醒。

## 2025-12-03

- 調整：`split_sentences`
  - 不再使用標點符號（如 `. ? !` 或全形對應）在行內切句。
  - 直接以原始換行為單位，每一行視為一個朗讀單位。
  - 避免誤切像「z.B.」這類包含句點的縮寫，減少不自然的斷句。

## 2025-12-04

- 調整：輸出類型預設值
  - 側邊欄的「輸出類型」`st.radio` 預設改為「產生黑底 MP4 影片」（`index=1`）。
  - 仍可手動改回「只產生 MP3」，但 README 現在也明確說明預設是黑底 MP4。
